## 1. 프로젝트 개요

### 제품명

**Asset Mate**

### 제품 정의

Adobe AEM Assets의 기능을 강화하는 Addon 플랫폼. AEM Assets가 기본적으로 제공하지 않거나 부족한 영역을 보완하여, 마케터와 외부 에이전시가 에셋을 더 효율적으로 검색, 검증, 협업, 변환, 배포할 수 있도록 지원한다.

### 배포 형태

**Hybrid** (독립 웹 애플리케이션 + AEM UI 내 임베디드 확장)

- **독립 웹앱**: 소셜 리사이징, 대시보드, 에이전시 포털 등 풀 기능 제공
- **AEM 임베디드**: 에셋 상세 뷰 내 빠른 리사이즈, 브랜드 검사 등 컨텍스트 기능

### 기술 스택 (목업)

- React 19 + TypeScript
- Vite (빌드 도구)
- React Spectrum S2 (`@react-spectrum/s2`) - Adobe 공식 디자인 시스템
- Style Macros (`unplugin-parcel-macros`)
- React Router (다중 페이지)

## 2. 타겟 사용자

### 주요 사용자 그룹

| 사용자 | 역할 | 주요 사용 기능 |
|--------|------|----------------|
| **마케터 / 콘텐츠 매니저** | 캠페인 기획, 에셋 관리, 소셜 배포 | 에셋 검색, 소셜 리사이징, 승인 워크플로, 배포 관리 |
| **외부 에이전시** | 크리에이티브 제작, 시안 제출 | 브랜드 가이드라인 확인, 에셋 다운로드, 시안 업로드/리뷰 |

### 사용자 니즈 분석

**마케터 / 콘텐츠 매니저**
- 캠페인에 맞는 에셋을 빠르게 찾고 싶다
- 에셋을 소셜 채널별로 한 번에 리사이즈하고 싶다
- 에이전시가 제출한 시안을 효율적으로 리뷰하고 승인하고 싶다
- 브랜드 가이드라인이 잘 지켜지고 있는지 확인하고 싶다
- 브랜드 위반 에셋을 디자이너 없이 AI로 빠르게 수정하고 싶다
- 자연어로 간단한 에셋 수정을 직접 하고 싶다
- 여러 채널에 동시에 배포하고 이력을 관리하고 싶다

**외부 에이전시**
- 클라이언트의 브랜드 가이드라인과 에셋을 쉽게 확인하고 싶다
- 제작한 시안에 대한 피드백을 빠르게 받고 싶다
- 시안의 브랜드 위반 사항을 AI가 자동으로 보정해주면 좋겠다
- 승인된 최종 에셋을 안전하게 다운로드하고 싶다

## 3. 핵심 주제 (6 Themes)

### 구현 순서 (사용자 여정 기반)

```
[1. 에셋 탐색] → [2. 브랜드 검증] → [3. AI 수정] → [4. 협업/승인] → [5. 채널별 변환] → [6. 최적화/전달]
     Search          Brand        AI Creative     Collaboration    Social Resize     Optimization
                                  (Firefly)
```

사용자가 에셋을 **찾고(Search)**, 브랜드 규정에 **적합한지 확인하고(Brand)**, 위반 사항을 **AI로 수정하고(AI Creative)**, 관련자들과 **리뷰/승인하고(Collaboration)**, 채널에 맞게 **변환하고(Social Resize)**, 최적의 품질로 **전달하는(Optimization)** 자연스러운 흐름을 따른다.

### Theme 1: Advanced Search & Discovery (1순위)

> AEM 기본 검색을 넘어서는 고급 에셋 탐색 기능

#### 기능 아이디어

| # | 기능 | 설명 | 우선도 |
|---|------|------|--------|
| 1-1 | 비주얼 유사도 검색 | 이미지를 드래그&드롭하면 AI가 시각적으로 유사한 에셋을 찾아줌 | Must |
| 1-2 | 색상 팔레트 검색 | 특정 색상 조합으로 에셋 필터링, 브랜드 컬러 기반 자동 매칭 | Must |
| 1-3 | 자연어 시맨틱 검색 | "가을 분위기 야외 라이프스타일 사진" 같은 자연어 쿼리 지원 | Must |
| 1-4 | 복합 필터 조합 | 채널 + 캠페인 + 시즌 + 포맷 + 언어를 조합한 파워 필터 | Must |
| 1-5 | 다국어 유사 에셋 추출 | 동일 캠페인의 다른 언어 버전 에셋을 자동으로 그룹핑 | Must |
| 1-6 | 중복 에셋 탐지 | 동일/유사 에셋 자동 감지 및 정리 제안 | Must |
| 1-7 | 텍소노미 기반 검색 | 조직의 분류 체계(텍소노미)를 기준으로 한 구조화된 검색 | Must |

#### 차별점 (vs AEM 기본 검색)

- AEM 기본: 키워드 기반 텍스트 검색 + 기본 메타데이터 필터
- Asset Mate: AI 비주얼 검색 + 시맨틱 검색 + 색상 검색 + 다국어 그룹핑 + 텍소노미 트리

### Theme 2: Brand Governance & Compliance (2순위)

> 브랜드 일관성과 컴플라이언스를 보장하는 거버넌스 기능

#### 기능 아이디어

| # | 기능 | 설명 | 우선도 |
|---|------|------|--------|

| 2-2 | 라이선스/저작권 만료 관리 | 스톡 이미지, 모델 초상권 등의 만료일 추적 및 알림 | Must |
| 2-3 | 브랜드 승인 게이트 | 미승인 에셋의 외부 배포 차단, 승인 상태 시각화 | Must |
| 2-4 | 템플릿 잠금 영역 | 에이전시에게 제공하는 템플릿에서 수정 불가 영역 설정 (로고 위치, 폰트 등) | Must |
| 2-5 | 브랜드 스코어카드 | 에셋별 브랜드 준수도 점수화 및 리포팅 | Must |
| 2-6 | 금지 에셋 관리 | 사용 금지된 에셋(만료, 리콜 등) 목록 관리 및 사용 시 경고 | Must |

#### 차별점 (vs AEM 기본)

- AEM 기본: 수동 승인 워크플로, 메타데이터 기반 상태 관리
- Asset Mate: AI 자동 브랜드 검사 + 스코어링 + 라이선스 추적 + 게이트 자동화

### Theme 3: AI Creative Assistant - Firefly (3순위)

> Adobe Firefly API를 활용한 AI 기반 에셋 편집 및 브랜드 자동 보정

#### 기능 아이디어

| # | 기능 | 설명 | 우선도 |
|---|------|------|--------|
| 3-1 | 브랜드 위반 자동 수정 | T2 브랜드 검사에서 발견된 위반 사항(색상/로고/배경/텍스트)을 Firefly가 자동 보정 | Must |
| 3-2 | 자연어 기반 편집 | "배경을 더 따뜻하게", "로고를 오른쪽 상단으로 이동" 등 자연어로 에셋 수정 | Must |
| 3-3 | Generative Fill | 불필요 요소 제거 후 AI로 자연스럽게 채우기 (배경 정리, 오브젝트 제거) | Must |
| 3-4 | Generative Expand | 소셜 리사이즈 시 부족한 영역을 AI로 자연스럽게 확장 (예: 1:1→9:16 변환) | Must |
| 3-5 | 브랜드 Custom Model | 브랜드 에셋으로 학습한 Firefly 커스텀 모델로 브랜드 톤 일관성 유지 | Must |
| 3-6 | 브랜드 변형 생성 | 기존 에셋 기반으로 시즌/무드/배경 변형을 AI로 자동 생성 | Must |

#### Firefly API 매핑

| 기능 | Firefly API | 설명 |
|------|------------|------|
| 브랜드 위반 수정 | Image-to-Image Instruct Edit (Image5) | 프롬프트로 기존 이미지 구조 유지하며 수정 |
| 자연어 편집 | Image-to-Image Instruct Edit (Image5) | 자연어 명령을 편집 프롬프트로 변환 |
| Generative Fill | Fill Image API | 마스킹 영역을 AI로 채우기 |
| Generative Expand | Expand Image API | 캔버스 확장 시 AI로 콘텐츠 생성 |
| Custom Model | Custom Models API (`/v3/images/generate-async`) | 브랜드 에셋 10-30개로 스타일 모델 학습 |
| 변형 생성 | Generate Image API + Custom Model | 커스텀 모델 기반 변형 이미지 생성 |

#### 브랜드 위반 자동 수정 상세

| 위반 유형 | AI 수정 방식 |
|-----------|-------------|
| 색상 보정 | 브랜드 컬러에 맞지 않는 색상 톤을 Instruct Edit로 자동 조정 |
| 로고 보정 | 로고 위치/크기/여백이 가이드라인에 맞도록 재배치 |
| 배경 보정 | 브랜드 톤에 맞지 않는 배경을 Generative Fill/Expand로 교체/조정 |
| 텍스트 보정 | 브랜드 폰트/크기/위치 가이드라인에 맞춰 재조정 |

#### 차별점

- AEM 기본: Firefly 연동 없음, 별도 Creative Cloud 앱에서 수동 편집 필요
- Asset Mate: DAM 내에서 바로 AI 편집 + 브랜드 검사 연계 자동 수정 + 커스텀 모델 브랜드 일관성

## 4. 핵심 사용자 시나리오

### 시나리오 1: 캠페인 런칭

```
마케터가 신규 캠페인 에셋을 검색
  → 적합한 에셋을 소셜 채널별로 리사이즈
  → 브랜드 검사 실행
  → 위반 사항이 있으면 Firefly AI 수정 요청
  → AI 수정 결과가 Inbox에 등록 (Pending)
  → 담당자가 Inbox에서 전/후 비교 후 Approve
  → 승인 요청 → 승인 완료 후 소셜 채널에 배포
```

**관련 Theme**: T1 → T5 → T2 → T3 → T4 → T5

### 시나리오 2: 에이전시 협업

```
마케터가 에이전시에 브랜드 에셋 패키지 공유
  → 에이전시가 시안 제작 후 업로드
  → 브랜드 자동 검사 → 위반 시 AI 수정 제안
  → 마케터가 시안 리뷰 (마크업/코멘트)
  → 수정 요청 또는 승인
```

**관련 Theme**: T4 → T2 → T3 → T4

### 시나리오 3: 브랜드 관리 + AI 보정

```
관리자가 전체 에셋의 브랜드 준수도 확인
  → 스코어카드로 위반 에셋 식별
  → 위반 에셋에 대해 Firefly AI 일괄 보정 요청
  → AI Creative Inbox에 결과 등록 (Pending)
  → Inbox에서 전/후 비교 확인 → 일괄 Approve / Reject
  → 라이선스 만료 임박 에셋 확인
```

**관련 Theme**: T2 → T3

### 시나리오 4: 다채널 배포

```
마케터가 원본 에셋 선택
  → 전체 소셜 채널 사이즈로 원클릭 리사이즈
  → 비율이 다른 채널은 Generative Expand로 AI 확장
  → 각 채널별 프리뷰 확인 및 미세 조정
  → 소셜 캘린더에서 배포 일정 설정
  → 일괄 배포 실행 및 이력 기록
```

**관련 Theme**: T1 → T5 → T3 → T6

### 시나리오 5: AI 기반 에셋 변형 생성 (신규)

```
마케터가 기존 캠페인 에셋 선택
  → "겨울 시즌 버전으로 변경" 자연어 입력
  → Firefly가 브랜드 Custom Model 기반으로 변형 생성
  → 여러 변형 후보 중 선택
  → 브랜드 검사 통과 확인 → 승인 프로세스 진행
```

**관련 Theme**: T3 → T2 → T4

## 7. 향후 확장 가능 영역 (Post-MVP)

브레인스토밍 과정에서 논의되었으나 MVP 범위에서 제외된 주제들:

- **Asset Intelligence & Analytics**: 에셋 사용 현황 분석, AI 기반 인사이트, 활용도/ROI 대시보드
- **Integration Hub**: 서드파티 도구 연동 (Figma, Canva, Slack 등), API 게이트웨이
- **Reporting & Executive Dashboard**: 팀 생산성, 에셋 퍼포먼스, 스토리지 사용량 등 경영 리포트
- **Security & Access Governance**: 세분화된 접근 제어, 감사 추적, 워터마크 자동 적용
- **Asset Lifecycle Management**: 버전 이력 시각화, 자동 아카이빙
