## 1. 공통 기능

### F-0.1 메인 대시보드

**유저 스토리**
- 마케터로서, 로그인 후 전체 현황을 한눈에 파악하고 싶다.
- 마케터로서, 자주 사용하는 기능에 빠르게 접근하고 싶다.

**상세 기능**

| 구성 요소 | 설명 |
|-----------|------|
| 현황 카드 | 전체 에셋 수, 승인 대기 건수, 만료 임박 라이선스, 예정된 배포 건수 |
| 최근 활동 피드 | 최근 업로드/승인/배포된 에셋 타임라인 |
| 빠른 액션 | 에셋 검색, 소셜 리사이즈, 리뷰 바로가기 버튼 |
| 캠페인 진행 현황 | 진행 중인 캠페인별 에셋 완료율 프로그레스 바 |
| 브랜드 건강도 | 전체 에셋 브랜드 준수율 게이지 |

**인수 조건**
- [ ] 대시보드 로딩 시 모든 현황 카드가 3초 내 표시된다
- [ ] 각 카드 클릭 시 관련 상세 화면으로 이동한다
- [ ] 최근 활동은 최근 7일간의 데이터를 표시한다

---

### F-0.2 에셋 상세 뷰

**유저 스토리**
- 마케터로서, 에셋의 상세 정보와 활용 가능한 액션을 한 곳에서 확인하고 싶다.

**상세 기능**

| 구성 요소 | 설명 |
|-----------|------|
| 미리보기 | 이미지/비디오 전체 화면 미리보기, 확대/축소 |
| 메타데이터 패널 | 파일 정보, 태그, 텍소노미, 캠페인, 언어, 채널 |
| 브랜드 스코어 | 브랜드 준수도 점수 및 검사 항목별 결과 |
| 버전 이력 | 버전 목록 및 이전 버전과의 비교 |
| 액션 바 | 다운로드, 리사이즈, 공유, 리뷰 요청, 승인 |
| 관련 에셋 | 유사 에셋, 다국어 버전, 동일 캠페인 에셋 |
| 라이선스 정보 | 라이선스 유형, 만료일, 사용 제한 조건 |

**인수 조건**
- [ ] 이미지 미리보기가 원본 해상도로 확대 가능하다
- [ ] 메타데이터 인라인 편집이 가능하다
- [ ] 액션 바에서 모든 주요 작업을 수행할 수 있다

### F-0.3 설정 허브 (`/settings`)

**유저 스토리**
- 브랜드 관리자로서, 브랜드 거버넌스 가이드를 한곳에서 만들고 게시해 업로드·AI 보정·대시보드 검사에 반영하고 싶다.
- 운영자로서, 텍소노미·채널 프리셋·알림 임계값 등을 설정에서 조정하고 관련 화면과 일관되게 유지하고 싶다.

**상세 기능**

| 구성 요소 | 설명 |
|-----------|------|
| **브랜드 거버넌스 가이드 (필수)** | 가이드 목록(이름·버전·적용 범위: 전역/캠페인·채널 오버라이드)·상태(초안/게시/예약). 규칙 세트 편집: 금지 요소, 로고 안전영역, 색·타이포, 필수 메타데이터, 라이선스·만료 정책 연동 여부. 위반 유형별 심각도 매핑(통과/경고/차단). 게시 시 유효일·감사 한 줄 이력(확장 시 전체 로그). |
| 텍소노미·분류 정책 | 분류 트리·필수 태그 정책 요약 및 검색/업로드와 연동되는 기대 동작 설명(고우선). |
| 채널·소셜 프리셋 | 채널별 출력 사양과 소셜 리사이즈 화면과의 정합(고우선). |
| 알림·임계값 | 라이선스 만료 N일 전 알림, 브랜드 위반 알림 등 브랜드 인박스·헤더 알림과 연동되는 임계값. |
| AI 사용 정책 | 자동 보정 허용 범위, 인간 승인 필수 여부, 프롬프트·로그 보존 기간(중우선). |
| 역할·사용자 | 관리자/편집자 등 역할 요약 및 사용자 목록 진입점(중우선). |
| 브랜드 하위 정책 링크 | 템플릿 잠금·금지 에셋 관리 화면으로의 딥링크(중우선). |
| 연동·검색 기본값·감사 로그 | AEM/DAM URL·동기화, 검색 기본 탭/건수, 감사 로그보내기(후순위·목업 토글 수준). |

**인수 조건**
- [ ] `/settings` 진입 시 **브랜드 거버넌스 가이드** 섹션이 기본으로 노출되고, 목록에서 가이드를 선택·편집·초안 저장·게시(목업)할 수 있다.
- [ ] 가이드에 적용 범위(전역 vs 캠페인/채널 오버라이드)와 심각도 매핑이 UI에 표시된다.
- [ ] 텍소노미·채널 프리셋·알림 영역이 구분되어 있으며, 각각 관련 기능 화면(검색·소셜·브랜드 인박스)과 연결되는 설명 또는 링크가 있다.
- [ ] AI 정책·사용자·템플릿 잠금/금지 에셋에 대한 설정 또는 진입 링크가 제공된다.

### F-0.4 AI Creative — 거버넌스 우선 생성·DAM 업로드 (`/assets/upload`)

**유저 스토리**

- 마케터로서, 신규 크리에이티브를 만들기 전에 브랜드 거버넌스에 맞는지 확인하고, 통과한 결과만 DAM에 넣고 싶다.
- DAM 관리자로서, 저장 경로를 탐색하거나 AI가 제안한 경로를 참고해 일관된 폴더 구조를 유지하고 싶다.

**상세 기능**

| 구성 요소 | 설명 |
|-----------|------|
| 세션 모드 | **이미지 채팅** 참조 업로드 + 대화로 생성·재생성 / **텍스트 전용** 자연어로 이미지 생성(목업 프리뷰) |
| 거버넌스 (목업) | 파일명·프롬프트 휴리스틱으로 통과·경고·차단; 차단 시 생성·업로드 확정 불가 |
| 세션 단계 | `awaiting_content` → 이미지 확정 후 `ready_to_upload` → `done`; 단계에 따라 우측 패널(조건·미리보기·CTA) 표시 |
| DAM 경로 | **`ready_to_upload`** 이고 거버넌스가 차단이 아닐 때만 노출. 경로 탐색(목 선택형) + **AI 제안 경로**(세션 유형에 따른 하위 폴더 접미) |
| 초기화 | 「조건·대화 초기화」 시 채팅·우측 패널을 검색 AI 탭과 유사한 **빈 초기 상태**(플레이스홀더)로 되돌림 |
| 레이아웃 | 2열 그리드(채팅 \| 적용 조건·결과); 우측 열은 `minmax(320px, 1fr)`로 최소 폭 확보 |

**인수 조건**

- [ ] `/assets/upload`에서 이미지·텍스트 전용 생성 플로가 구분되어 동작한다(목업).
- [ ] 거버넌스 차단 시 생성·DAM·하단 확정 CTA가 비활성화되거나 숨겨진다.
- [ ] DAM 경로 블록은 **이미지가 생성되어 `ready_to_upload` 단계**일 때만 나타난다.
- [ ] 초기화 후 우측에 칩·요약·미리보기·DAM·하단 CTA가 나타나지 않는다(신규 세션 전).

## 2. Theme 1: Advanced Search & Discovery

### F-1.1 비주얼 유사도 검색

**유저 스토리**
- 마케터로서, 참조 이미지와 비슷한 느낌의 에셋을 찾고 싶다.
- 마케터로서, 기존 캠페인 에셋과 톤앤매너가 유사한 에셋을 발견하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 입력 방식 | 이미지 드래그&드롭, 파일 업로드, 기존 에셋에서 "유사 에셋 찾기" 버튼 |
| 검색 알고리즘 | 이미지 임베딩 기반 코사인 유사도 매칭 |
| 결과 표시 | 유사도 점수(%) 순 정렬, 그리드/리스트 뷰 전환 |
| 필터 결합 | 비주얼 유사도 결과에 채널/캠페인/포맷 필터 추가 적용 |
| 유사도 임계값 | 슬라이더로 유사도 기준 조절 (예: 80% 이상만 표시) |

**인수 조건**
- [ ] 드래그&드롭으로 이미지 업로드 후 3초 내 유사 에셋 결과 표시
- [ ] 유사도 임계값 조절 시 결과가 실시간으로 필터링된다
- [ ] 결과에서 에셋 클릭 시 상세 뷰로 이동한다

---

### F-1.2 색상 팔레트 검색

**유저 스토리**
- 마케터로서, 특정 색상 톤의 에셋을 찾아 캠페인 일관성을 유지하고 싶다.
- 마케터로서, 브랜드 컬러에 맞는 에셋만 필터링하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 색상 선택 | 컬러 피커, HEX 코드 직접 입력, 브랜드 컬러 프리셋 |
| 다중 색상 | 최대 5개 색상 조합으로 검색 |
| 색상 비중 | 각 색상의 비중(dominance) 조절 슬라이더 |
| 브랜드 컬러 매칭 | 등록된 브랜드 컬러 팔레트에서 원클릭 검색 |
| 결과 표시 | 에셋별 추출된 주요 색상 팔레트 오버레이 |

**인수 조건**
- [ ] 컬러 피커에서 색상 선택 후 즉시 결과가 업데이트된다
- [ ] 브랜드 컬러 프리셋 클릭 시 해당 색상 조합으로 자동 검색된다
- [ ] 에셋 카드에 추출된 주요 색상이 표시된다

---

### F-1.3 자연어 시맨틱 검색

**유저 스토리**
- 마케터로서, "봄 시즌 아웃도어 라이프스타일"처럼 자연어로 에셋을 검색하고 싶다.
- 마케터로서, 키워드를 정확히 몰라도 의도를 설명하면 적합한 에셋을 찾고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 입력 | 자유 텍스트 검색창 (자연어 입력 가이드 제공) |
| AI 해석 | 입력된 자연어를 분석하여 시맨틱 벡터로 변환 |
| 검색 범위 | 이미지 내용 + 메타데이터 + 태그를 종합적으로 매칭 |
| 검색 제안 | 입력 중 관련 검색어/태그 자동 완성 |
| 결과 설명 | 각 결과가 왜 매칭되었는지 하이라이트 표시 |

**인수 조건**
- [ ] "가을 분위기 카페 인테리어" 같은 자연어 입력 시 관련 에셋이 반환된다
- [ ] 검색 결과에 매칭 이유가 툴팁으로 표시된다
- [ ] 검색어 자동 완성이 입력 300ms 후 표시된다

---

### F-1.4 복합 필터 조합

**유저 스토리**
- 마케터로서, 여러 조건을 동시에 적용하여 정확한 에셋을 빠르게 찾고 싶다.

**상세 기능**

| 필터 카테고리 | 옵션 |
|--------------|------|
| 채널 | Instagram, Facebook, YouTube, TikTok, LinkedIn, X, 디스플레이 |
| 캠페인 | 캠페인명 목록 (검색 가능 드롭다운) |
| 시즌 | Spring, Summer, Fall, Winter, Year-round |
| 포맷 | Image (JPG, PNG, WebP), Video (MP4, MOV), PDF, PSD, AI |
| 언어 | 전체 언어 목록 (다국어 사이트 기반) |
| 승인 상태 | 승인됨, 검토 중, 수정 요청, 미제출 |
| 에셋 유형 | 사진, 일러스트, 아이콘, 배너, 로고, 비디오 |
| 날짜 범위 | 업로드일, 수정일, 만료일 기준 |

**인수 조건**
- [ ] 필터 적용 시 실시간으로 결과 건수가 업데이트된다
- [ ] 적용된 필터가 태그 형태로 검색창 아래 표시되며 개별 제거 가능하다
- [ ] 필터 조합을 저장하여 재사용할 수 있다 (저장된 필터)
- [ ] 모든 필터를 한 번에 초기화하는 "필터 초기화" 버튼이 있다

---

### F-1.5 다국어 유사 에셋 추출

**유저 스토리**
- 마케터로서, 동일 캠페인의 다른 언어 버전 에셋을 한눈에 확인하고 싶다.
- 마케터로서, 특정 에셋의 전체 언어 버전 완성 현황을 파악하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 자동 그룹핑 | 동일 캠페인 내 에셋을 언어 태그 기반으로 자동 그룹핑 |
| 그룹 뷰 | 원본 에셋 + 언어별 변형 에셋을 행렬 형태로 표시 |
| 완성도 지표 | 언어별 제작 현황 (완료/진행중/미착수) 시각화 |
| 누락 알림 | 특정 언어 버전이 누락된 경우 경고 표시 |
| 비교 모드 | 두 언어 버전을 나란히 비교하여 차이 확인 |

**인수 조건**
- [ ] 에셋 선택 시 "다국어 버전 보기" 액션으로 그룹 뷰 진입 가능
- [ ] 각 언어별 완성 상태가 색상 코드로 구분된다
- [ ] 누락된 언어 버전에 대한 제작 요청을 바로 생성할 수 있다

---

### F-1.6 중복 에셋 탐지

**유저 스토리**
- DAM 관리자로서, 중복 에셋을 찾아 스토리지를 효율적으로 관리하고 싶다.
- 마케터로서, 같은 에셋이 여러 버전으로 존재할 때 어떤 것이 최신인지 확인하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 탐지 범위 | 완전 동일 (해시 비교) + 유사 (비주얼 유사도 기반) |
| 결과 그룹핑 | 중복/유사 에셋을 그룹으로 묶어 표시 |
| 비교 뷰 | 그룹 내 에셋들을 나란히 비교 (해상도, 날짜, 사이즈) |
| 정리 액션 | 보존할 에셋 선택, 나머지 아카이브/삭제 제안 |
| 스캔 스케줄 | 주기적 자동 스캔 또는 수동 실행 |

**인수 조건**
- [ ] 중복 그룹별로 "원본 유지" 에셋을 선택할 수 있다
- [ ] 정리 실행 전 영향 범위 (참조 중인 페이지 등) 확인 가능
- [ ] 탐지 결과를 리포트로 다운로드할 수 있다

### F-1.7 텍소노미 기반 검색

**유저 스토리**
- 마케터로서, 조직의 분류 체계를 탐색하면서 관련 에셋을 찾고 싶다.
- 마케터로서, 카테고리 트리를 통해 에셋을 체계적으로 브라우징하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 트리 네비게이션 | 텍소노미 계층 구조를 트리 뷰로 탐색 |
| 브레드크럼 | 현재 위치를 브레드크럼으로 표시 |
| 필터 결합 | 텍소노미 선택과 다른 필터(채널, 언어 등) 결합 |
| 에셋 카운트 | 각 카테고리 노드별 에셋 수 표시 |
| 태그 기반 연결 | 에셋의 태그와 텍소노미 노드 간 매핑 관리 |

**인수 조건**
- [ ] 텍소노미 트리에서 카테고리 클릭 시 해당 에셋이 오른쪽 패널에 표시된다
- [ ] 각 노드에 에셋 수가 실시간으로 표시된다
- [ ] 복수 노드 선택(체크박스) 시 교집합/합집합 전환 가능

### F-1.8 대화형 AI 검색 (멀티모달 오케스트레이션)

**유저 스토리**

- 마케터로서, 채팅으로 “캠페인·색·텍소노미·참조 이미지”를 한 흐름에서 말하며 에셋을 찾고 싶다.
- 마케터로서, 시스템이 해석한 조건을 칩으로 확인하고 잘못된 조건만 빼고 다시 검색하고 싶다.

**F-1.3 자연어 시맨틱 검색과의 차이**

| 항목 | F-1.3 시맨틱 | F-1.8 AI 검색 |
|------|----------------|----------------|
| 입력 | 단발 자연어 | 멀티턴 대화 + 규칙/향후 LLM 파싱 |
| 범위 | 시맨틱 매칭 중심 | 필터·비주얼 유사·색상·텍소노미를 **의도 스키마**로 조합 |
| 투명성 | 매칭 이유 표시 | 적용 조건 칩 + 설명 메시지 |

**상세 기능**

| 항목 | 설명 |
|------|------|
| 대화 UI | 사용자/어시스턴트 메시지 스레드, 전송·초기화 |
| 의도 모델 | `SearchIntent`: 텍스트, 캠페인, 채널, 색(HEX), 텍소노미 prefix, 참조 에셋 ID, 유사도 하한 등 |
| 실행 | 활성 조건 **교집합**으로 에셋 필터(목업은 키워드 규칙, 실서비스는 structured output 등으로 교체) |
| 결과 | 기존 검색과 동일한 에셋 카드·유사도/매칭 이유 표시 |
| 조건 편집 | 칩별 제거, 「조건·대화 초기화」 |
| 결과 영역 표시 정책 | **첫 AI 메시지 전송이 성공한 뒤에만** 건수·요약·결과 그리드를 표시한다. 전송 전·초기화 직후에는 플레이스홀더(안내)만 표시한다 |

**인수 조건**

- [ ] 메시지 전송 시 파싱된 조건이 칩과 어시스턴트 요약에 반영된다
- [ ] 참조 에셋 ID가 있으면 유사도 하한 이상만 결과에 포함된다
- [ ] 첫 성공 전송 전에는 결과 그리드·건수가 보이지 않고 안내 플레이스홀더가 보인다
- [ ] 「조건·대화 초기화」 후 결과 영역이 다시 플레이스홀더 상태로 돌아간다

## 3. Theme 2: Brand Governance & Compliance

**유저 스토리**

## 4. Theme 3: AI Creative Assistant (Firefly)

### F-3.1 브랜드 위반 자동 수정

**유저 스토리**
- 마케터로서, 브랜드 검사에서 위반된 에셋을 디자이너에게 보내지 않고 AI로 바로 수정하고 싶다.
- 에이전시로서, 시안 제출 전에 브랜드 위반 사항을 AI가 자동으로 보정해주면 좋겠다.
- DAM 관리자로서, AI가 수정한 결과를 승인 전까지 원본에 반영하지 않고 Inbox에서 확인하고 싶다.

**워크플로**

```
브랜드 검사 위반 감지
  → AI 수정 요청 (개별 or 일괄)
  → Firefly AI 수정 실행 → 수정 후보 생성
  → AI Creative Inbox에 결과 등록 (상태: 대기)
  → 담당자가 Inbox에서 전/후 비교 확인
  → Approve: 수정본이 원본을 대체 → 브랜드 재검사 자동 실행
  → Reject: 수정 폐기, 위반 상태 유지 (수동 수정 필요)
  → Request Changes: 다른 후보 선택 or 프롬프트 조정 후 재생성
```

**상세 기능**

| 항목 | 설명 |
|------|------|
| 연계 트리거 | T2 브랜드 검사 결과에서 "AI 자동 수정" 버튼으로 진입 |
| 색상 보정 | 브랜드 컬러에 맞지 않는 색상 톤을 Firefly Instruct Edit로 자동 조정 |
| 로고 보정 | 로고 위치/크기/여백을 가이드라인에 맞도록 재배치 |
| 배경 보정 | 브랜드 톤에 맞지 않는 배경을 Generative Fill로 교체/조정 |
| 텍스트 보정 | 폰트/크기/위치를 브랜드 가이드에 맞춰 조정 |
| 전/후 비교 | 수정 전/후를 Side-by-Side로 비교, 변경 부분 하이라이트 |
| 수정 후보 | 각 위반 항목별 최대 3개의 수정 후보 제안 |
| 일괄 수정 | 다수 에셋의 동일 유형 위반을 일괄로 AI 수정 |
| **AI Creative Inbox** | AI 수정 결과를 리스트로 모아 보여주는 대기 큐 |
| **승인 워크플로** | Approve / Reject / Request Changes 3단계 |
| Firefly API | Image-to-Image Instruct Edit (Image5), Fill Image API |

**AI 수정 Creative 상세**

| 항목 | 설명 |
|------|------|
| Inbox 진입점 | `/ai/inbox` 직접 URL·브랜드 대시보드 위젯·**헤더 알림(벨)** 대화 등 (목업). 사이드바 **AI Creative**는 `/assets/upload` 거버넌스 생성 허브로, Inbox와는 별도 |
| 리스트 항목 | 에셋 썸네일, 위반 유형, 수정 일시, 요청자, 상태 배지 |
| 상태 | `대기(Pending)` → `승인(Approved)` / `거절(Rejected)` / `수정요청(Changes Requested)` |
| 필터/정렬 | 상태별, 위반 유형별, 요청자별, 날짜별 필터 |
| 배지 카운트 | **헤더 벨** 아이콘에 미처리 건수 배지 표시 (목업) |
| 일괄 승인 | 체크박스로 다건 선택 후 일괄 Approve/Reject |
| 상세 진입 | 리스트 항목 클릭 시 전/후 비교 상세 화면으로 이동 |

**승인 워크플로 상세**

| 액션 | 동작 |
|------|------|
| **Approve** | 수정본이 원본을 대체, 브랜드 재검사 자동 실행, 버전 이력에 "AI 수정" 기록 |
| **Reject** | 수정 폐기, 원본 유지, 위반 상태 그대로, 사유 입력 필수 |
| **Request Changes** | 다른 후보 선택 또는 프롬프트 조정 후 재생성, 상태가 다시 Pending으로 |

**인수 조건**
- [ ] 브랜드 검사 결과 화면에서 "AI 수정" 버튼이 위반 항목 옆에 표시된다
- [ ] AI 수정 완료 후 결과가 Inbox에 Pending 상태로 등록된다
- [ ] Inbox에서 상태별/유형별 필터링이 가능하다
- [ ] 수정 후보 최소 2개 이상이 제안된다
- [ ] 전/후 비교에서 변경 부분이 시각적으로 하이라이트된다
- [ ] Approve 시 수정본이 원본을 대체하고 브랜드 재검사가 자동 실행된다
- [ ] Reject 시 사유 입력이 필수이며 원본이 유지된다
- [ ] Request Changes 시 후보 재선택 또는 재생성이 가능하다
- [ ] 일괄 수정/일괄 승인 시 진행률이 표시된다
- [ ] 헤더 알림 등을 통해 미처리 Inbox 건수를 확인하고 Inbox로 이동할 수 있다

### F-3.2 자연어 기반 편집

**유저 스토리**
- 마케터로서, "배경을 더 따뜻하게", "하늘을 맑게 변경" 같은 자연어로 에셋을 수정하고 싶다.
- 마케터로서, 디자인 도구를 몰라도 텍스트 입력만으로 에셋을 편집하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 프롬프트 입력 | 자연어 편집 명령 입력 필드 (예시 프롬프트 제공) |
| 편집 프리뷰 | 명령 입력 후 실시간 프리뷰 (원본과 나란히) |
| 편집 이력 | 연속 편집 명령의 이력 관리, 특정 단계로 되돌리기 |
| **AI 편집 실행 이력 보기** | **「AI 생성 이력 보기」로 다이얼로그를 열어, 이전/다음·`n/N` 슬라이드로 단계별 이미지·목업 편집 프롬프트를 확인.「이 이미지로 교체」로 해당 단계를 스튜디오 편집 프리뷰·편집 명령 필드에 반영** |
| 프롬프트 제안 | 자주 사용하는 편집 명령 템플릿 제공 |
| 브랜드 가드레일 | 편집 결과가 브랜드 가이드를 벗어나면 경고 표시 |
| Firefly API | Image-to-Image Instruct Edit (Image5) |

**예시 프롬프트**

| 카테고리 | 예시 |
|----------|------|
| 색상/톤 | "전체적으로 더 따뜻한 톤으로", "채도를 높여줘", "빈티지 느낌으로" |
| 배경 | "배경을 흐리게", "배경을 사무실로 변경", "하늘을 맑은 날씨로" |
| 오브젝트 | "왼쪽 나무 제거", "배경에 건물 추가", "사람 뒤 배경 정리" |
| 스타일 | "좀 더 프로페셔널하게", "밝고 활기찬 느낌으로", "미니멀하게 정리" |
| 구도 | "피사체를 중앙으로 이동", "여백을 더 넓게", "크롭을 더 타이트하게" |

**인수 조건**
- [ ] 자연어 입력 후 10초 내 편집 프리뷰가 표시된다
- [ ] 편집 이력에서 특정 단계로 되돌리기가 가능하다
- [ ] **AI 편집 실행 이력 보기**로 다이얼로그에서 슬라이드·`n/N`으로 단계를 탐색할 수 있고, **이 이미지로 교체** 시 해당 이미지·프롬프트가 편집 프리뷰 영역에 반영된다
- [ ] 브랜드 가이드 위반 시 경고가 표시되고 위반 항목이 명시된다
- [ ] 프롬프트 템플릿에서 원클릭으로 적용 가능하다

### F-3.3 Generative Fill (AI 채우기)

**유저 스토리**
- 마케터로서, 에셋에서 불필요한 요소를 제거하고 자연스럽게 채워넣고 싶다.
- 마케터로서, 에셋의 특정 영역을 다른 콘텐츠로 교체하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 마스킹 도구 | 브러시, 올가미, 사각 선택 등으로 영역 마스킹 |
| 제거 모드 | 선택 영역의 오브젝트를 제거하고 배경으로 자연스럽게 채움 |
| 교체 모드 | 선택 영역을 텍스트 프롬프트 기반으로 새 콘텐츠로 교체 |
| 프리뷰 | 채우기 결과 프리뷰, 최대 4개 후보 |
| 브랜드 모델 | Custom Model 적용 시 브랜드 스타일 유지 |
| Firefly API | Fill Image API |

**인수 조건**
- [ ] 브러시 도구로 영역 선택 후 "제거" 또는 "교체" 모드 선택 가능
- [ ] 교체 모드에서 프롬프트 입력 후 최소 2개 후보 제안
- [ ] 결과가 원본과 자연스럽게 블렌딩된다

### F-3.4 Generative Expand (AI 확장)

**유저 스토리**
- 마케터로서, 소셜 리사이즈 시 비율이 다른 채널용으로 부족한 영역을 AI로 자연스럽게 확장하고 싶다.
- 마케터로서, 가로 이미지를 세로(9:16)로 변환할 때 잘리는 부분 없이 확장하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| T5 연계 | 소셜 리사이즈에서 비율 변환 시 자동으로 Expand 제안 |
| 확장 방향 | 상/하/좌/우 개별 또는 전체 방향 확장 |
| 타겟 사이즈 | 채널 프리셋 사이즈에 맞춰 자동 확장 |
| 프리뷰 | 확장 결과 프리뷰, 원본 영역과 확장 영역 구분 표시 |
| 프롬프트 가이드 | 확장 영역에 대한 추가 프롬프트 (예: "같은 배경 패턴으로") |
| Firefly API | Expand Image API |

**인수 조건**
- [ ] 1:1 이미지를 9:16으로 변환 시 AI 확장이 자동 제안된다
- [ ] 확장 영역과 원본 영역의 경계가 자연스럽다
- [ ] 소셜 리사이즈 플로우에서 원클릭으로 Expand를 적용할 수 있다

### F-3.5 브랜드 Custom Model

**유저 스토리**
- DAM 관리자로서, 브랜드 에셋으로 Firefly 커스텀 모델을 학습시켜 브랜드 일관성을 유지하고 싶다.
- 마케터로서, AI 편집/생성 시 항상 우리 브랜드 톤에 맞는 결과를 얻고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 학습 데이터 선택 | AEM Assets에서 브랜드 대표 에셋 10-30개 선택 |
| 모델 유형 | Photography Style, Illustration Style 중 선택 |
| 학습 실행 | Firefly Custom Models API로 학습 실행 (비동기) |
| 모델 관리 | 학습된 모델 목록 관리, 활성/비활성 전환 |
| 기본 모델 설정 | AI 편집/생성 시 기본 적용할 Custom Model 지정 |
| 결과 비교 | 기본 모델 vs Custom Model 결과 비교 |
| Firefly API | Custom Models API (`/v3/images/generate-async`) |

**인수 조건**
- [ ] AEM에서 에셋 선택 후 "Custom Model 학습" 워크플로 시작 가능
- [ ] 학습 진행 상태가 표시된다 (예상 소요 시간 포함)
- [ ] 학습 완료된 모델이 AI 편집/생성 시 선택 가능하다
- [ ] Custom Model 적용 유무에 따른 결과 비교가 가능하다

### F-3.6 브랜드 변형 생성

**유저 스토리**
- 마케터로서, 기존 에셋을 기반으로 시즌별/무드별 변형을 빠르게 만들고 싶다.
- 마케터로서, 하나의 원본 에셋에서 여러 캠페인용 변형을 생성하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 원본 선택 | 에셋 상세 뷰에서 "변형 생성" 실행 |
| 변형 유형 | 시즌 (봄/여름/가을/겨울), 무드 (밝은/차분한/역동적), 배경 변경 |
| 프롬프트 입력 | "여름 시즌 해변 버전으로" 같은 자연어로 변형 방향 지정 |
| 후보 생성 | 최대 4개 변형 후보를 한 번에 생성 |
| Custom Model | 브랜드 Custom Model 적용으로 브랜드 일관성 유지 |
| 선택 & 저장 | 후보 중 선택, 원본과의 관계 자동 설정 |
| 브랜드 검사 | 생성된 변형에 대한 자동 브랜드 검사 |
| Firefly API | Generate Image API + Custom Models API |

**인수 조건**
- [ ] 변형 유형/프롬프트 입력 후 4개 후보가 생성된다
- [ ] Custom Model 적용 시 브랜드 톤이 일관되게 유지된다
- [ ] 선택한 변형이 원본과 자동으로 관계 설정된다
- [ ] 생성된 변형에 대해 자동 브랜드 검사가 실행된다

## 5. Theme 4: Collaboration & Approval Workflow

### F-4.1 에셋 공유 포털

**유저 스토리**
- 마케터로서, 에이전시에게 브랜드 에셋을 안전하게 패키지로 전달하고 싶다.
- 에이전시로서, 클라이언트의 최신 브랜드 에셋을 한 곳에서 확인하고 다운로드하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 에셋 패키지 | 에셋을 선택하여 공유 패키지로 구성 |
| 공유 링크 | 만료 기간, 다운로드 횟수 제한 설정 가능한 링크 생성 |
| 브랜드 킷 | 로고, 컬러, 폰트, 가이드라인 문서를 브랜드 킷으로 구성하여 제공 |
| 접근 로그 | 누가 언제 어떤 에셋을 다운로드했는지 추적 |
| 버전 동기화 | 에셋 업데이트 시 에이전시에게 자동 알림 |

**인수 조건**
- [ ] 공유 링크 생성 시 만료일/다운로드 횟수 설정 가능
- [ ] 에이전시가 AEM 계정 없이도 공유 링크로 에셋 다운로드 가능
- [ ] 다운로드 이력이 관리자에게 실시간 표시된다

### F-4.2 리뷰/피드백 루프

**유저 스토리**
- 마케터로서, 에이전시가 제출한 시안 위에 직접 마크업하고 피드백을 남기고 싶다.
- 에이전시로서, 피드백을 명확히 확인하고 수정 버전을 제출하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 비주얼 마크업 | 이미지 위에 직접 그리기, 화살표, 영역 표시, 텍스트 주석 |
| 코멘트 스레드 | 마크업 포인트별 코멘트 스레드 (대화형) |
| 워크플로 상태 | 제출 → 검토 중 → 수정 요청 → 재제출 → 승인 |
| 알림 | 상태 변경/새 코멘트 시 이메일 + 인앱 알림 |
| 이력 | 전체 리뷰 이력 (코멘트, 상태 변경, 버전) 타임라인 |

**인수 조건**
- [ ] 이미지 위에 포인트 지정 후 코멘트를 남길 수 있다
- [ ] 각 코멘트에 "해결됨" 상태를 설정할 수 있다
- [ ] 워크플로 상태 변경 시 관련자에게 자동 알림 발송

---

### F-4.3 버전 비교

**유저 스토리**
- 마케터로서, 에셋의 수정 전/후를 나란히 비교하여 변경 사항을 확인하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|

| 오버레이 뷰 | 두 버전을 겹쳐서 슬라이더로 비교 |
| 차이점 하이라이트 | 변경된 영역을 자동 감지하여 하이라이트 |
| 메타데이터 비교 | 해상도, 파일 크기, 색상 프로필 등 비교 |
| 버전 선택 | 전체 버전 이력에서 임의의 두 버전 선택 |

**인수 조건**
- [ ] Side-by-Side 뷰에서 두 이미지의 스크롤/줌이 동기화된다
- [ ] 오버레이 슬라이더로 변경 부분을 직관적으로 확인 가능
- [ ] 버전 목록에서 변경자, 변경일, 변경 사유가 표시된다

### F-4.4 캠페인 워크스페이스

**유저 스토리**
- 마케터로서, 캠페인별로 에셋, 태스크, 일정을 한 곳에서 관리하고 싶다.

**인수 조건**

- [ ] 타임라인에서 태스크 일정을 드래그로 조정 가능

### F-4.5 승인 상태 대시보드

**유저 스토리**

- [ ] 배치 선택 후 일괄 승인/거절 실행 가능

## 6. Theme 5: Social Resizing & Distribution

### F-5.1 원클릭 멀티 리사이즈

**유저 스토리**
- 마케터로서, 하나의 원본 에셋에서 모든 소셜 채널 사이즈를 한 번에 생성하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 원본 선택 | 에셋 상세 뷰 또는 검색 결과에서 "소셜 리사이즈" 실행 |
| 채널 선택 | 체크박스로 생성할 채널/사이즈 선택 |
| 전체 선택 | "전체 채널" 원클릭으로 모든 사이즈 일괄 선택 |
| 프리뷰 그리드 | 선택된 모든 사이즈의 리사이즈 결과를 한 화면에 프리뷰 |
| 미세 조정 | 개별 사이즈별 크롭 위치, 확대/축소 조정 |
| 일괄 생성 | 확인 후 모든 사이즈 에셋을 일괄 생성 및 저장 |

**인수 조건**
- [ ] 원클릭으로 최소 10개 이상의 채널 사이즈가 동시 생성 가능
- [ ] 프리뷰에서 각 채널의 실제 비율로 결과를 확인 가능
- [ ] 개별 리사이즈 결과의 크롭 포인트를 드래그로 조정 가능
- [ ] 생성된 에셋이 원본과 자동으로 연결(관계) 설정된다

---

### F-5.2 채널별 프리셋 관리

**유저 스토리**
- 마케터로서, 각 소셜 채널의 최신 사이즈 규격을 관리하고 커스텀 사이즈를 추가하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 기본 프리셋 | 주요 소셜 채널 (Instagram, Facebook, YouTube 등) 표준 사이즈 내장 |
| 커스텀 프리셋 | 사용자 정의 이름, 너비, 높이, DPI 설정 |
| 프리셋 그룹 | 채널별, 캠페인별, 팀별로 프리셋 그룹핑 |
| 즐겨찾기 | 자주 사용하는 프리셋 즐겨찾기 설정 |
| 업데이트 알림 | 채널의 공식 사이즈 변경 시 알림 |

**인수 조건**
- [ ] 기본 프리셋에 모든 주요 소셜 채널 표준 사이즈가 포함되어 있다
- [ ] 커스텀 프리셋 생성/편집/삭제가 가능하다
- [ ] 프리셋에 채널 아이콘과 용도 설명이 함께 표시된다

---

### F-5.3 스마트 리사이즈 프리뷰

**유저 스토리**
- 마케터로서, 리사이즈 결과가 각 소셜 채널에서 실제로 어떻게 보일지 미리 확인하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 디바이스 프레임 | 각 채널의 실제 UI 프레임 내에서 에셋 미리보기 |
| 탭 전환 | 채널 탭으로 빠르게 전환하며 확인 |
| 비교 모드 | 원본과 리사이즈 결과를 나란히 비교 |
| 안전 영역 | 텍스트/로고가 잘리지 않는 안전 영역 가이드라인 표시 |
| 실시간 조정 | 크롭 위치 조정 시 프리뷰가 실시간 업데이트 |

**인수 조건**
- [ ] Instagram 피드, 스토리 등 채널 UI 프레임이 정확히 표현된다
- [ ] 안전 영역 가이드가 토글로 on/off 가능
- [ ] 크롭 조정 시 0.5초 내 프리뷰가 업데이트된다

---

### F-5.4 소셜 캘린더

**유저 스토리**
- 마케터로서, 소셜 미디어 배포 일정을 캘린더로 관리하고 예약 배포하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 캘린더 뷰 | 월간/주간/일간 뷰 전환 |
| 드래그 스케줄링 | 에셋을 캘린더의 날짜/시간에 드래그&드롭 |
| 채널 레인 | 채널별 레인으로 구분된 타임라인 뷰 |
| 배포 상태 | 예정 / 배포 중 / 완료 / 실패 상태 표시 |
| 반복 스케줄 | 정기 배포 패턴 설정 (매주, 매월 등) |

**인수 조건**
- [ ] 캘린더에서 에셋 클릭 시 상세 정보 팝오버 표시
- [ ] 드래그&드롭으로 배포 일정 변경 가능
- [ ] 채널별 색상 코딩으로 구분된다

---

### F-5.5 다채널 동시 배포

**유저 스토리**
- 마케터로서, 승인된 에셋을 여러 소셜 채널에 동시에 배포하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 채널 선택 | 배포 대상 채널 복수 선택 |
| 배포 옵션 | 즉시 배포 / 예약 배포 선택 |
| 캡션/설명 | 채널별 맞춤 캡션 및 해시태그 입력 |
| 배포 확인 | 배포 전 최종 확인 (프리뷰 + 메타데이터) |
| 결과 알림 | 배포 성공/실패 결과 알림 |

**인수 조건**
- [ ] 3개 이상의 채널에 동시 배포가 가능하다
- [ ] 채널별 캡션을 개별적으로 설정할 수 있다
- [ ] 배포 실패 시 재시도 옵션이 제공된다

### F-5.6 배포 이력 추적

**유저 스토리**
- 마케터로서, 어떤 에셋이 어느 채널에 언제 배포되었는지 추적하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 이력 테이블 | 에셋명, 채널, 배포일, 배포자, 상태 목록 |
| 필터 | 채널별, 기간별, 상태별 필터 |
| 에셋 연결 | 이력에서 원본 에셋 및 리사이즈 에셋으로 이동 |
| 통계 | 채널별 배포 건수, 기간별 추이 차트 |
| 내보내기 | 이력 데이터 CSV/Excel 다운로드 |

**인수 조건**
- [ ] 기간 필터로 특정 기간의 배포 이력 조회 가능
- [ ] 에셋별 배포 이력이 에셋 상세 뷰에서도 확인 가능
- [ ] 통계 차트에서 채널별/기간별 배포 추이 확인 가능

## 7. Theme 6: Asset Optimization & Delivery

### F-6.1 자동 포맷 변환

**유저 스토리**
- 마케터로서, 에셋을 채널에 최적화된 포맷으로 자동 변환하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 지원 포맷 | JPG, PNG, WebP, AVIF, GIF, SVG |
| 채널 맞춤 | 채널별 권장 포맷으로 자동 선택 |
| 품질 설정 | 품질 레벨 조절 (파일 크기 vs 화질 밸런스) |
| 비교 프리뷰 | 변환 전/후 화질 비교 + 파일 크기 표시 |
| 배치 변환 | 다수 에셋 일괄 포맷 변환 |

**인수 조건**
- [ ] 변환 전/후 파일 크기 차이가 표시된다
- [ ] 품질 슬라이더 조절 시 실시간으로 프리뷰 업데이트
- [ ] 배치 변환 시 진행률 표시

---

### F-6.2 스마트 크롭

**유저 스토리**
- 마케터로서, AI가 주요 피사체를 인식하여 채널별로 최적의 크롭을 자동으로 적용하면 좋겠다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 피사체 인식 | AI가 이미지 내 주요 피사체/얼굴/텍스트 영역 감지 |
| 자동 크롭 | 피사체를 중심으로 각 사이즈에 최적 크롭 자동 적용 |
| 수동 보정 | 자동 크롭 결과를 사용자가 드래그로 미세 조정 |
| 포커스 포인트 | 수동으로 포커스 포인트 지정 (추후 리사이즈에 적용) |

**인수 조건**
- [ ] 얼굴이 포함된 이미지에서 얼굴이 잘리지 않도록 크롭된다
- [ ] 포커스 포인트 지정 시 모든 리사이즈에 반영된다
- [ ] 자동 크롭과 수동 보정 간 전환이 원클릭으로 가능

---

### F-6.3 품질/용량 최적화

**유저 스토리**
- 마케터로서, 채널별 권장 해상도와 용량에 맞게 에셋을 자동 최적화하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 채널 프로필 | 채널별 권장 해상도, 최대 용량, DPI 프로필 |
| 자동 최적화 | 선택된 채널에 맞춰 해상도/용량 자동 조절 |
| 품질 비교 | 최적화 전/후 화질 비교 (확대 뷰) |
| 용량 표시 | 최적화 후 예상 파일 크기 미리 표시 |

**인수 조건**
- [ ] 채널 선택 시 권장 사양이 자동 적용된다
- [ ] 최적화 후 원본 대비 용량 절감률이 표시된다

---

### F-6.4 배치 다운로드

**유저 스토리**
- 마케터로서, 여러 에셋을 한 번에 선택하고 채널별 사양으로 일괄 다운로드하고 싶다.

**상세 기능**

| 항목 | 설명 |
|------|------|
| 다수 선택 | 검색 결과/캠페인 에셋에서 다수 에셋 체크박스 선택 |
| 다운로드 옵션 | 원본 / 특정 채널 사양 / 전체 채널 사이즈 |
| 파일 구조 | 채널별 폴더 구조로 ZIP 패키징 |
| 네이밍 규칙 | 파일명 규칙 설정 (에셋명_채널_사이즈.확장자) |
| 다운로드 이력 | 다운로드 일시, 사용자, 포맷 기록 |

**인수 조건**
- [ ] 10개 이상의 에셋을 동시에 선택하여 다운로드 가능
- [ ] ZIP 파일 내 채널별 폴더 구조가 올바르게 생성된다
- [ ] 다운로드 진행률이 표시된다
